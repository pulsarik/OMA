# Путеводитель по проекту

Этот проект - маленькое веб-приложение для дружеских раздач Omaha Hi-Lo: открыть стол, раздать карты, сохранить расклад и потом обсудить его с друзьями.

## Что есть в проекте

```text
Oma/
  server/        Серверная часть: логика раздачи, WebSocket, API, SQLite
  demo/          Демонстрационная часть: клиент, README, Playwright, описание проекта
    client/      Веб-интерфейс на React + Vite
    playwright/  UI/E2E тесты
    bots/        Черновик тестового бота
```

## Главные папки

### `server`

Это backend, то есть серверная часть.

Сервер:

- раздает карты;
- принимает WebSocket-сообщения от клиента;
- сохраняет раздачи в SQLite;
- отдает список сохраненных раздач через HTTP.

Важные файлы:

- `server/src/game.ts` - создание колоды, перемешивание, раздача Omaha-рук.
- `server/src/index.ts` - запуск HTTP-сервера и WebSocket-сервера.
- `server/src/handStore.ts` - сохранение и загрузка раздач из SQLite.
- `server/tests/game.spec.ts` - тест игровой логики.
- `server/package.json` - список зависимостей и команд сервера.

### `demo/client`

Это frontend, то есть веб-страница, которую ты открываешь в браузере.

Клиент:

- показывает кнопки и состояние игры;
- подключается к серверу через WebSocket;
- отправляет команду `deal`;
- получает события от сервера.

Важные файлы:

- `demo/client/index.html` - HTML-точка входа. В ней есть `<div id="root">`, куда React вставляет приложение.
- `demo/client/src/main.tsx` - запускает React-приложение.
- `demo/client/src/pages/App.tsx` - главный экран приложения.
- `demo/client/package.json` - зависимости и команды клиента.

### `demo`

Это вспомогательная папка вокруг демо-приложения.

Важные файлы:

- `demo/README.md` - как установить и запустить проект.
- `demo/project-description.md` - цель проекта и план развития.
- `demo/package.json` - общие команды, например запуск сервера и клиента вместе.
- `demo/playwright.config.ts` - настройки браузерных тестов.

## Что такое Node.js

Node.js - это программа, которая умеет запускать JavaScript вне браузера.

Обычно JavaScript живет в браузере: Chrome, Edge, Firefox. Но серверную часть тоже можно писать на JavaScript или TypeScript. Для этого и нужен Node.js.

В этом проекте Node.js запускает:

- сервер в папке `server`;
- инструменты сборки клиента;
- тесты;
- команды из `package.json`.

## Что такое npm

npm - это менеджер пакетов для Node.js.

Он делает две главные вещи:

- скачивает зависимости в папку `node_modules`;
- запускает команды из `package.json`.

Примеры:

```powershell
npm.cmd run install:all
npm.cmd run dev
npm.cmd test
```

На Windows иногда PowerShell блокирует `npm`, поэтому можно использовать `npm.cmd`.

## Что такое TypeScript

TypeScript - это JavaScript с типами.

Например, TypeScript помогает заранее поймать ошибку вида: "у этой функции нет такого поля" или "сюда передали не тот тип данных".

Файлы TypeScript обычно имеют расширения:

- `.ts` - обычный TypeScript;
- `.tsx` - TypeScript с React-разметкой.

Перед запуском в браузере или Node.js TypeScript превращается в обычный JavaScript.

## Что такое React

React - это библиотека для создания интерфейса.

Вместо ручного создания HTML через строки мы пишем компоненты. Например `App.tsx` описывает, какие кнопки, заголовки и блоки должны быть на странице.

В этом проекте React отвечает за веб-страницу игры.

## Что такое Vite

Vite - это dev server и сборщик frontend-приложения.

Он:

- запускает локальный сайт, обычно на `http://localhost:5173`;
- быстро обновляет страницу при изменении кода;
- собирает готовую версию клиента командой `npm.cmd --prefix demo/client run build`.

## Что такое Express

Express - это библиотека для HTTP-сервера в Node.js.

В этом проекте Express отвечает за адреса вроде:

```text
GET http://localhost:4000/admin/hands
GET http://localhost:4000/admin/hands/:id
```

То есть через Express браузер или тесты могут запросить сохраненные раздачи.

## Что такое WebSocket

WebSocket - это постоянное соединение между браузером и сервером.

Обычный HTTP работает так: клиент спросил, сервер ответил, соединение закончилось.

WebSocket работает иначе: клиент подключился, и сервер может отправлять события в любой момент. Для игры это удобно, потому что состояние стола должно обновляться сразу у всех игроков.

В этом проекте клиент отправляет серверу:

```json
{ "action": "deal", "players": 2 }
```

А сервер отвечает событием:

```json
{ "type": "hand_dealt", "data": { "...": "..." } }
```

## Что такое SQLite

SQLite - это маленькая база данных в одном файле.

Не нужно ставить отдельный сервер базы данных. Приложение просто создает файл и хранит там данные.

В этом проекте SQLite нужна для сохранения раздач, чтобы потом открыть их заново.

## Как проект запускается

Из папки `demo`:

```powershell
cd C:\Users\test1234\Documents\Oma\demo
npm.cmd run dev
```

Эта команда запускает сразу две части:

- сервер: `http://localhost:4000`;
- клиент: обычно `http://localhost:5173`.

Открывать в браузере нужно клиент:

```text
http://localhost:5173
```

Серверный адрес `http://localhost:4000` нужен не как красивая страница, а как API для клиента.

## Как думать об архитектуре

Проще всего так:

```text
Браузер с React
  |
  | WebSocket / HTTP
  v
Node.js сервер
  |
  | читает и пишет
  v
SQLite база с раздачами
```

Браузер показывает интерфейс. Сервер принимает команды и хранит состояние. SQLite сохраняет историю раздач.
