# Omaha Hi-Lo Replay Project

## Цель проекта
Сделать простое веб-приложение для дружеских раздач в Omaha Hi-Lo (8 or better): собраться с друзьями за одним виртуальным столом, раздать карты, сохранить интересный расклад, потом открыть его заново и спокойно обсудить.

Проект не задумывается как казино, турнирная система или строгая покерная платформа. Главная ценность - удобный домашний инструмент для игры, повторов и разбора спорных или красивых раздач.

## Основные сценарии

### Живая дружеская раздача
- Несколько игроков подключаются к одному столу.
- Каждый игрок видит свои 4 закрытые карты.
- Общие карты открываются по улицам: flop, turn, river.
- Можно быстро начать новую раздачу без сложной регистрации и лишних настроек.

### Сохранение раскладов
- Каждая раздача сохраняется с уникальным идентификатором.
- Сохраняются руки игроков, общие карты, время создания и служебные данные для повтора.
- У интересной раздачи должна быть ссылка или ID, который можно отправить друзьям.

### Повтор и обсуждение
- Сохраненную раздачу можно открыть позже.
- В режиме повтора можно смотреть борд и руки игроков.
- Нужны переключатели: показать все карты, скрыть чужие карты, показать итог.
- Важен удобный просмотр high/low результата: кто забирает high, есть ли low, как делится банк.

### Режим ведущего / просмотра
- Нужна отдельная страница для просмотра всех сохраненных раздач.
- На ней можно открыть любую раздачу полностью, включая скрытые карты.
- Этот режим нужен не для безопасности, а для разбора, споров, обучения и отладки.

### Боты и тестовые игроки
- Боты не являются главной целью первой версии.
- Полезный минимум: быстро заполнить стол тестовыми игроками и нагенерировать раздачи.
- Позже можно добавить простую логику ботов, чтобы сравнивать решения и обсуждать варианты.

## Игровая логика
- Реализовать раздачу Omaha: 4 закрытые карты каждому игроку и 5 общих карт.
- Реализовать правила Omaha Hi-Lo (8 or better).
- Для оценки комбинации игрок должен использовать ровно 2 карты из руки и ровно 3 карты с борда.
- Определять high-победителя.
- Определять low-победителя, если есть qualifying low 8-or-better.
- Показывать деление банка между high и low сторонами.

## Интерфейс
- Минимальный, но удобный стол для друзей.
- Видимые карты должны быть легко читаемыми.
- Нужны понятные действия: новая раздача, открыть следующую улицу, сохранить, открыть повтор.
- Для обсуждения важнее ясность расклада, чем декоративный дизайн.

## Сеть и хранение
- Сервер на Node.js + TypeScript.
- WebSocket для обновления состояния стола в реальном времени.
- SQLite для хранения сохраненных раздач.
- React + TypeScript для веб-интерфейса.

## Тестирование
- Unit-тесты для раздачи и оценки Omaha Hi-Lo.
- Тесты на сохранение и повтор раздач.
- Минимальные Playwright-тесты для основных пользовательских сценариев.

## Приоритеты реализации
1. Привести базовую структуру проекта в рабочее состояние.
2. Сделать надежную модель раздачи: колода, игроки, борд, стадии hand/flop/turn/river/showdown.
3. Добавить сохранение и загрузку раздач.
4. Сделать страницу повтора раздачи.
5. Реализовать оценку high и low по правилам Omaha Hi-Lo.
6. Улучшить интерфейс стола для дружеской игры.
7. Добавить режим просмотра всех раздач.
8. Добавить тестовых игроков или простых ботов.

## Примечание
Проект на текущем этапе - это домашний прототип для игры с друзьями и обсуждения раскладов. Поэтому решения должны быть простыми, понятными и удобными для живого использования: меньше турнирной бюрократии, больше ясного показа карт, повторов и результатов.
